## screenpipe

AI memory for your screen. Record. Search. Automate. All local, all private, all yours.

[screenpi.pe](https://screenpi.pe) · [docs](https://docs.screenpi.pe) · [discord](https://discord.gg/screenpipe) · [x](https://twitter.com/screen_pipe)

---

## values

**local-first always**
Data never leaves your device unless you explicitly opt in. No account required. No cloud dependency. If it doesn't work offline, it's not done.

**open source by default**
Trust is earned through transparency. MIT licensed. Fully auditable. The code is the contract.

**user fanatic**
Build what people want and bring maximum value. Users who stay are obsessed. Users who leave hit bugs. Fix what's broken before building what's new.

**positive sum**
Give first. Optimize for the long term. We're all going to win — it's a multiplayer, positive sum game.

**radical transparency**
No ego. Idea meritocracy. Constructive disagreement. Truth over comfort.

**bold over safe. ship over plan.**
Small, focused changes. Every commit should be deployable. If you can't measure it, you can't improve it.

**humans augmented by AI**
Humans should focus on profound exploration, not repetitive digital labor. AI needs full context to truly automate. Screen is the universal interface — we build the ambient layer between humans and their digital work.

---

## principles

- simplicity: less is more. less code, less files, less dependencies, less complexity
- stability over features: fix what's broken before building what's new
- no feature creep: every feature must serve record, rewind, or ask
- cross-platform: if it doesn't work on macOS, Windows, and Linux, it's not done
- respect the user's machine: CPU, memory, disk — screenpipe runs 24/7. performance is not optional
